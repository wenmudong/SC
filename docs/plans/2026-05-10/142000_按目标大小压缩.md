# 图片压缩工具 — 按目标大小压缩

## Context

用户希望除了「按质量压缩」外，还能「按目标大小压缩」——输入指定 KB/MB，将每张图片压缩到该大小以下。原图已小于目标的跳过不压缩。

## 核心设计

### 后端 API 变更

`POST /api/tools/compress` 新增参数：
- `compress_mode: str = Form("quality")` — `"quality"`（按质量）或 `"target_size"`（按目标大小）
- `target_size_kb: int = Form(0)` — 目标大小（KB），仅 `compress_mode="target_size"` 时生效

**按目标大小压缩算法**（二分查找 quality）：
1. 先检查原图是否已小于目标 → 跳过
2. 用 quality=50 压缩一次，检查大小
3. 二分查找 quality（1-100），找到能满足目标大小的最高 quality
4. 即使 quality=1 仍超目标，用 quality=1 输出（尽力而为）

### 前端 UI 变更

控制区新增「压缩模式」选择：
- `按质量`：显示现有的质量滑块（不变）
- `按目标大小`：显示数字输入 + 单位下拉（KB/MB），隐藏质量滑块

## 后端详细变更 — `tools.py`

新增辅助函数：
```python
def _compress_to_target_size(img, save_format, target_bytes, original_bytes):
    """二分查找 quality，将图片压缩到目标大小以下"""
    if original_bytes <= target_bytes:
        return None  # 已小于目标，跳过

    # 二分查找最优 quality
    low, high = 1, 100
    best_buf = None
    while low <= high:
        mid = (low + high) // 2
        buf = io.BytesIO()
        img.save(buf, format=save_format, quality=mid)
        if len(buf.getvalue()) <= target_bytes:
            best_buf = buf
            low = mid + 1  # 尝试更高质量
        else:
            high = mid - 1  # 降低质量

    # 如果 quality=1 仍超目标，用 quality=1
    if best_buf is None:
        best_buf = io.BytesIO()
        img.save(best_buf, format=save_format, quality=1)

    best_buf.seek(0)
    return best_buf
```

修改 `compress_images` 函数：
- 当 `compress_mode == "target_size"` 时，调用 `_compress_to_target_size`
- 返回 None 表示跳过该文件（不写入 zip）
- 需要先将图片完整转为 bytes 才能做二分，因为每次 save 需要重新编码

## 测试用例

| 测试 | 输入 | 预期 |
|------|------|------|
| target_size 基本功能 | 大图 + target=50KB | 200, zip 中文件 < 50KB |
| target_size 跳过小图 | 小图(1KB) + target=100KB | 200, zip 为空（跳过） |
| target_size 混合场景 | 大图+小图 + target=50KB | 200, zip 只含大图 |
| target_size 质量=1 仍超 | 极小 target=1KB | 200, 尽力压缩 |
| compress_mode 无效值 | compress_mode="invalid" | 400 |
