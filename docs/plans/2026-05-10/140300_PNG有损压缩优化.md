# 图片压缩工具 — 统一有损压缩优化

## Context

图片压缩工具已基本实现，但用户发现 PNG 是无损格式，quality 参数对 PNG 无效。需要修改为「统一有损压缩」—— 所有格式都经过有损压缩，quality 参数对所有格式都生效。

## 核心变更

**PNG 处理逻辑变更**：

| 模式 | JPG | PNG | WebP |
|------|-----|-----|------|
| 保持原格式（改前） | JPG + quality | PNG 无损（quality 无效） | WebP + quality |
| 保持原格式（改后） | JPG + quality | **转为 JPG** + quality（有损） | WebP + quality |
| 转 WebP | WebP + quality | WebP + quality | WebP + quality |

> PNG 在「保持原格式」模式下会转为 JPG（因为 PNG 本身不支持有损压缩），文件后缀从 `.png` 变为 `.jpg`。PNG 的 alpha 通道（透明度）会被转为白色背景。

## 修改内容

### 1. 后端 — `sc-backend/app/routers/tools.py`

修改 `_convert_to_rgb_if_needed` 和压缩逻辑中 PNG 的处理：

**当前逻辑**（`output_format == "original"` 时）：
```python
save_format = TYPE_TO_FORMAT[file.content_type]  # PNG → "PNG"
out_name = _sanitize_filename(file.filename)       # photo.png
```

**改为**：
```python
if file.content_type == "image/png":
    save_format = "JPEG"                    # PNG 转为 JPG（有损）
    out_name = Path(file.filename).stem + ".jpg"  # photo.jpg
else:
    save_format = TYPE_TO_FORMAT[file.content_type]
    out_name = _sanitize_filename(file.filename)
```

PNG 转 JPG 时需要：
- RGBA → RGB（丢弃 alpha 通道，白色背景）
- P（调色板）→ RGB
- CMYK → RGB

### 2. 后端测试 — `sc-backend/tests/routers/test_compress.py`

新增/修改测试用例：
- **新增** `test_compress_png_to_jpg_original` — PNG + format=original → zip 中为 `.jpg`（验证有损压缩生效）
- **修改** `test_compress_single_png_to_webp` — 保持不变（PNG + format=webp → .webp）
- **新增** `test_compress_quality_affects_png` — 验证 quality=1 和 quality=100 对 PNG 的压缩结果大小有差异

### 3. 前端 — `sc-frontend/src/app/tools/compress/page.tsx`

- 更新格式选择的说明文字，让用户清楚「保持原格式」模式下 PNG 会转为 JPG
- 格式选择区域添加提示：`"PNG 文件将转为 JPG 以实现有损压缩"`

### 4. 前端 — `sc-frontend/src/app/tools/compress/page.tsx` 文件列表

- 当文件是 PNG 且模式为「保持原格式」时，格式标签显示为 `PNG→JPG`，提示用户该文件会被转换
