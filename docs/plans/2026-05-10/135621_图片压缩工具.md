# 图片压缩工具实现计划

## Context

用户需要在 SuperCenter 网站中添加一个图片压缩工具，支持批量上传图片（含文件夹拖入），压缩后打包为 zip 下载，且压缩后的文件名与原文件一致。

## 需求总结

- **工具形态**：后端 API（FastAPI 压缩）+ 前端独立页面 `/tools/compress`
- **支持格式**：JPG、PNG、WebP
- **压缩质量**：用户可调 1-100 滑块
- **格式处理**：用户可选「保持原格式」或「统一转 WebP」
- **文件名**：压缩后保持原文件名不变
- **下载方式**：纯流式 zip 返回，不落盘，用户点击下载
- **权限**：需要登录

## 架构设计

```
前端 /tools/compress              后端
┌──────────────────────┐         ┌─────────────────────────┐
│ 拖拽区(文件/文件夹)    │  POST   │ POST /api/tools/compress │
│ 文件列表预览          │ ──────→ │                         │
│ 质量滑块 1-100        │         │ 1. 校验 content_type     │
│ 格式选择(原格式/WebP)  │         │ 2. Pillow 压缩           │
│ 压缩 & 下载按钮       │  ←──── │ 3. zipfile 打包 → 流式返回│
│        zip blob       │         └─────────────────────────┘
└──────────────────────┘
```

## 实施步骤

### 第一步：后端依赖 & 配置

1. **添加 Pillow 依赖**：`sc-backend/pyproject.toml` 添加 `"pillow>=10.0.0"`，执行 `uv sync`
2. **添加配置项**：`sc-backend/app/config.py` 新增：
   - `maxCompressSize: int = 20 * 1024 * 1024`（单文件 20MB 限制）
   - `maxCompressTotalSize: int = 500 * 1024 * 1024`（总大小 500MB 限制）

### 第二步：TDD — 先写测试（RED）

**新文件**：`sc-backend/tests/routers/test_compress.py`

测试用例：
| 测试 | 输入 | 预期 |
|------|------|------|
| 无认证 | 无 token | 401 |
| 单张 JPG 保持格式 | 1 JPEG, quality=80, format=original | 200, zip 含 1 个 .jpg |
| 单张 PNG 转 WebP | 1 PNG, quality=75, format=webp | 200, zip 含 1 个 .webp |
| 多文件混合 | 1 JPG + 1 PNG + 1 WebP | 200, zip 含 3 个文件 |
| 质量边界 | quality=1 / quality=100 | 200 |
| 无效质量 | quality=0 / quality=101 | 422 |
| 无效格式 | .txt 文件 | 400 |
| 文件名保留 | "photo_2024.jpg" | zip 中为 "photo_2024.jpg" |
| WebP 转换改名 | "photo.png" + format=webp | zip 中为 "photo.webp" |
| 文件过大 | 25MB 文件（限制 20MB） | 400 |

### 第三步：实现后端路由（GREEN）

**新文件**：`sc-backend/app/routers/tools.py`

```
router = APIRouter(prefix="/api/tools", tags=["工具"])

POST /compress
  - files: List[UploadFile] = File(...)
  - quality: int = Form(80, ge=1, le=100)
  - output_format: str = Form("original")  # "original" | "webp"
  - current_user: User = Depends(get_current_user)
  - 返回: StreamingResponse(media_type="application/zip")
```

压缩逻辑：
- `PIL.Image.open()` 打开图片
- 根据 output_format 选择保存格式
- JPEG + RGBA/P 模式 → 自动转 RGB
- CMYK → 自动转 RGB
- 文件名冲突 → 追加计数器（photo_1.jpg, photo_2.jpg）
- 文件名清理 → `Path(filename).name` 去除路径注入

**修改文件**：`sc-backend/app/main.py` — 注册 tools_router

### 第四步：前端 API 层

**修改文件**：`sc-frontend/src/services/api.ts`

新增 `compressApi.uploadAndCompress()` — 使用 raw fetch（非 request helper），返回 Blob：
- 构建 FormData（files + quality + output_format）
- 处理 401 → 触发 AUTH_EXPIRED_EVENT
- 处理错误 → 抛出中文错误信息

### 第五步：前端页面

**新文件**：`sc-frontend/src/app/tools/compress/page.tsx`

UI 布局（从上到下）：
1. `PageHeader` — 标题 "compress." + 描述
2. 拖拽区 — 虚线边框，支持拖拽文件/文件夹 + 点击选择
3. 文件列表 — 文件名、大小、格式标签、删除按钮
4. 控制区 — 质量滑块（1-100 + 数字显示）+ 格式选择（单选：保持原格式 / 转 WebP）
5. 压缩按钮 — 无文件或压缩中时禁用
6. 下载 — 压缩完成后自动触发 zip 下载

关键实现：
- `useAuthGuard()` 做登录校验
- 拖拽文件夹：`webkitGetAsEntry()` 递归遍历目录
- 下载：`URL.createObjectURL(blob)` + 隐藏 `<a>` 触发下载 + `revokeObjectURL` 释放内存

### 第六步：更新工具入口

**修改文件**：`sc-frontend/src/app/tools/page.tsx`

> **注意**：不修改顶部导航栏（Navbar.tsx）。通过在 /tools 页面的工具卡片列表中添加一个压缩工具卡片来引导用户进入。

在 TOOLS 数组中添加压缩工具条目，`linkUrl: "/tools/compress"`，`external: false`（内部链接用 Next.js Link）。

## 边界情况处理

| 场景 | 处理方式 |
|------|---------|
| CMYK 图片 | 自动转 RGB |
| 调色板模式 PNG | 自动转 RGB/RGBA |
| 文件名冲突（同名文件） | 追加计数器 |
| 文件名路径注入 | `Path.name` 清理 |
| 空文件 / 损坏图片 | try/except → 400 中文错误 |
| 前端超大文件 | 上传前校验总大小（500MB）和数量（50个），超限提示 |
| Token 过期 | 401 → 触发登录弹窗 |
| 内存占用 | 单文件 20MB + 总 500MB 限制 + 最多 50 个文件 |
