#### 实践清单

**1.1 断言精准化**
```python
# ❌ 模糊断言 - 难以定位问题
assert response.status_code == 200

# ✅ 精确断言 - 失败时一目了然
assert response.status_code == 200, f"Expected 200, got {response.status_code}. Body: {response.json()}"

# ✅ 结构化断言 - 验证数据完整性
assert response.json() == {
    "code": 0,
    "data": {
        "user_id": "10001",
        "name": "张三",
        "balance": Decimal("100.00")  # 金额用Decimal避免精度问题
    }
}
```

**1.2 数据真实性**
```
测试数据级别：
├── Mock数据 ────── 单元测试、快速验证
├── Fixture数据 ─── 集成测试、可重复性高
└── 真实数据 ────── E2E测试、预发布验证 (需脱敏)
```

**1.3 环境一致性**
```yaml
# docker-compose.yml 确保测试环境一致
services:
  app:
    image: ${APP_IMAGE}:${TAG}
    environment:
      - ENV=test
      - DB_HOST=db-test
    depends_on:
      - db-test
      - redis-test

  db-test:
    image: postgres:15
    environment:
      - POSTGRES_DB=testdb

  redis-test:
    image: redis:7-alpine
```

#### 常见错误与对策

| 错误类型 | 症状 | 对策 |
|---------|------|------|
| 隐式类型转换 | `"123" == 123` 通过 | 使用严格断言 |
| 浮点数比较 | `0.1 + 0.2 != 0.3` | 用 Decimal 或 tolerance |
| 时区问题 | 跨环境时间不一致 | 统一用 UTC，存储时间戳 |
| 并发竞态 | 偶发失败 | 显式等待 + 重试机制 |

**3.1 结构化日志**

```python
import logging
import json
from typing import Optional

class TestLogger:
    """测试日志结构化输出"""

    @staticmethod
    def log_test_step(step: str, context: dict, error: Optional[str] = None):
        """统一的测试步骤日志格式"""
        log_entry = {
            "event": "test_step",
            "step": step,
            "timestamp": datetime.now().isoformat(),
            "context": context,
        }
        if error:
            log_entry["error"] = {
                "type": type(error).__name__,
                "message": str(error),
            }
        print(json.dumps(log_entry, ensure_ascii=False))

    @staticmethod
    def log_api_response(response, request_id: str):
        """API响应日志"""
        print(json.dumps({
            "event": "api_response",
            "request_id": request_id,
            "status": response.status_code,
            "body": response.json() if response.content else None,
            "headers": dict(response.headers),
        }, ensure_ascii=False))
```

**4.2 并发能力测试**

```python
import pytest
import asyncio
from concurrent.futures import ThreadPoolExecutor

class TestConcurrency:
    """并发测试"""

    @pytest.mark.asyncio
    async def test_concurrent_requests(self):
        """模拟100并发请求"""
        async def make_request():
            async with aiohttp.ClientSession() as session:
                async with session.get('http://api/test') as resp:
                    return await resp.json()

        tasks = [make_request() for _ in range(100)]
        responses = await asyncio.gather(*tasks)

        assert len(responses) == 100
        # 验证无竞态条件
        assert len(set(responses)) > 1  # 结果多样性

    def test_thread_safety(self):
        """线程安全测试"""
        counter = Counter()  # 共享资源

        def increment():
            for _ in range(10000):
                counter.inc()

        with ThreadPoolExecutor(max_workers=10) as executor:
            futures = [executor.submit(increment) for _ in range(10)]
            for f in futures:
                f.result()

        assert counter.value == 100000, f"Expected 100000, got {counter.value}"
```

**5.1 数据脱敏**

```python
import hashlib
import re
from typing import Any

class DataSanitizer:
    """测试数据脱敏工具"""

    @staticmethod
    def mask_phone(phone: str) -> str:
        """手机号脱敏: 138****5678"""
        if not phone or len(phone) < 7:
            return "***"
        return f"{phone[:3]}****{phone[-4:]}"

    @staticmethod
    def mask_id_card(id_card: str) -> str:
        """身份证脱敏: 110***********1234"""
        if not id_card or len(id_card) < 8:
            return "***"
        return f"{id_card[:3]}**********{id_card[-4:]}"

    @staticmethod
    def mask_email(email: str) -> str:
        """邮箱脱敏: t***@example.com"""
        if not email or '@' not in email:
            return "***"
        local, domain = email.split('@', 1)
        return f"{local[0]}***@{domain}"

    @staticmethod
    def hash_for_compare(value: str, salt: str = "") -> str:
        """用于比对但不暴露原始值"""
        return hashlib.sha256(f"{value}{salt}".encode()).hexdigest()[:16]
```

# test_selector.py - 根据代码变动智能选择测试
class TestSelector:
    def __init__(self, git_diff):
        self.changed_files = git_diff

    def select_tests(self):
        """根据变更文件选择相关测试"""
        test_map = {
            "src/users/": ["tests/test_users.py", "tests/test_user_api.py"],
            "src/orders/": ["tests/test_orders.py", "tests/test_checkout.py"],
            "src/payment/": ["tests/test_payment.py", "tests/test_orders.py"],
        }

        selected = set()
        for changed in self.changed_files:
            for prefix, tests in test_map.items():
                if changed.startswith(prefix):
                    selected.update(tests)
        return list(selected)

│     ├── 闰年/闰秒                                           │

## 四、落地检查清单

### 测试设计Checklist

```
□ 1. 准确性
  □ 断言包含有意义的错误信息
  □ 使用精确断言而非模糊匹配
  □ 测试数据与环境一致
  □ 有明确的预期结果

□ 2. 覆盖度
  □ 需求用例全部覆盖
  □ 关键路径有测试
  □ 边界条件已覆盖
  □ 异常流程有测试

□ 3. 可观测性
  □ 失败时有完整日志
  □ 截图/视频已保存
  □ trace_id 可关联
  □ 报告可定位问题

□ 4. 性能
  □ 单测试 < 5s
  □ 支持并行执行
  □ 无内存泄漏
  □ 有超时保护

□ 5. 安全性
  □ 敏感数据已脱敏
  □ 测试环境隔离
  □ 无硬编码凭证
  □ 清理测试数据

□ 6. 效率
  □ 使用 Page Object / Fixture
  □ 避免重复代码
  □ 支持并行
  □ 可增量执行
```

## 六、总结

> **测试开发者的核心价值 = 置信度 × 效率**
>
> - **置信度** = 准确性 × 覆盖度 × 可观测性
> - **效率** = 自动化率 × 复用率 × 并行度
>
> 六个维度相互支撑：准确是底线，覆盖是广度，可观测是定位能力，性能是效率保障，安全是合规要求，效率是价值体现。
